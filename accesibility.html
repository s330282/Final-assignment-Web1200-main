Ensuring accessibility

We want to ensure that this website has high accessibility. This essentially means that it will be accessible (or usable) by as many people as possible. A lot of people have some sort of physical impairment which affects the way they engage with websites. Those who have poor eyesight may use screen readers, which is software that scans the text on a website and reads it out loud (text to speech). For this reason, we made sure that no text is contained in image files, which would render a screen reader useless.
Secondly, we ensured that it is possible to use the tab button to cycle through the navigation bar. This way, those with impaired vision are not forced to try to use the mouse or trackpad to navigate the website, which can obviously be hard or flat out impossible for them.
Color blindness is also a type of visual impairment we took into consideration when designing this website. Colorblind people typically struggle to separate green and red, so we avoided using those colors as much as possible.

We also wanted to make sure that the website is functional regardless of the type of device the user may choose to access it with. To achieve this, we added some code that checks the user’s device screen width and loads mobile CSS if the screen width is below a set limit.
Additionally, the website Achecker was used to look for ways to improve accessibility. Achecker is an automated tool that scans your code and alerts you if there are things that would hinder accessibility. Luckily for us, we only had one alert across all five webpages. The “error” was that a few images were missing an alt attribute. If an image for whatever reason cannot be loaded, the user instead sees a description of the image. It’s easy to see how this ties in to accessibility; even though an image does not load, the user isn’t left completely in the dark because they are told what the image depicts.
